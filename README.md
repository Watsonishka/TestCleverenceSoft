### Общий комментарий к выполнению ТЗ

При выполнении тестового задания я приняла решение не выделять отдельные библиотеки классов для каждого задания, чтобы не перегружать структуру проекта и не усложнять его навигацию. Все классы, относящиеся к решению, были размещены непосредственно в проектах соответствующих консольных приложений — Task1ConsoleApp, Task2ConsoleApp и Task3ConsoleApp. Это позволило сохранить простоту и прозрачность структуры: каждый проект полностью самодостаточен и содержит всё необходимое для выполнения конкретной задачи, без лишних зависимостей и пересечений между заданиями. Тестовый проект при этом ссылается на соответствующие консольные проекты для проверки их функциональности.

### Задача 1: Компрессия строк

В рамках технического задания была реализована компрессия и декомпрессия только для строк, состоящих из латинских букв. В связи с этим в Programm.cs была добавлена проверка входной строки на допустимые символы с помощью отдельного валидатора. На данный момент решение хорошо подходит для обработки кириллицы и других символов. Однако алгоритм плохо работает с цифрами, так как корректная декомпрессия строк, содержащих цифры, в данной реализации невозможна: невозможно однозначно определить, где в строке заканчиваются служебные данные о количестве повторений и начинается исходный текст. В рамках технического задания было принято решение не усложнять реализацию и не додумывать поведение для пограничных случаев.

### Задача 2: Многопоточный сервер

В рамках реализации многопоточного сервера для корректного тестирования пришлось добавить метод Reset(), который очищает состояние сервера и обнуляет счётчик перед запуском каждого теста, чтобы избежать влияния предыдущих тестов на последующие. Также от себя я добавила обработку переполнения целочисленного значения. Теперь при попытке прибавить значение, которое выводит счётчик за пределы int, выбрасывается исключение с информативным сообщением. Это позволяет избежать молчаливого переполнения и делает поведение сервера более предсказуемым.

### Задача 3: Парсер логов

При решении данной задачи я ориентировалась исключительно на примеры входных данных, поэтому написала два разных метода для двух разных форматов, отталкиваясь от их структуры и используя символ вертикальной черты как маркер формата. В техническом задании обнаружилось противоречие между примерами выходных данных, где дата выводилась в формате ГГГГ-ММ-ДД, и описанием требований к выходному формату, где был указан DD-MM-YYYY (день-месяц-год). Я выбрала реализовывать последнее, так как это было явно прописано в тексте требований. Также для корректного тестирования сохранения невалидных строк в выходной файл problems.txt был добавлен метод DeleteFile(), позволяющий очищать файл перед каждым запуском тестов и обеспечивающий изолированность тестов друг от друга.

### Тесты

Для удобства организации тестов я не стала создавать три отдельных тестовых проекта под каждое задание, а разместила все тесты в одном проекте с использованием xUnit. Чтобы избежать путаницы и сохранить чёткую структуру, тесты для каждой задачи были выделены в отдельные классы: Task1Tests для задач компрессии и декомпрессии строк, Task2Tests для тестирования многопоточного сервера и Task3Tests для проверки парсера логов. Такой подход позволил сохранить порядок, упростил навигацию по тестам и не перегружал решение лишними проектами.
